# 특허 발명설명서

## 발명의 명칭
**AI 기반 성경 원어 분석 및 개인화된 읽기 계획 제공 시스템 및 방법**

## 기술분야
본 발명은 인공지능 및 자연어 처리 기술을 활용한 디지털 학습 플랫폼에 관한 것으로, 더 구체적으로는 성경 원어(히브리어, 그리스어)를 실시간으로 분석하고, 사용자별 읽기 이력을 기반으로 개인화된 읽기 계획을 생성하며, 진행률을 추적하고 통계를 제공하며, 사용자 간 상호작용을 통한 커뮤니티 학습 경험을 제공하는 시스템 및 방법에 관한 것이다.

## 배경기술

### 기존 기술의 문제점

#### 1. 개인화된 학습 경험 제공의 복잡성
기존 시스템은 정적 학습 계획을 제공하여 다음과 같은 한계가 있다:

- **개인별 맞춤화 부족**: 모든 사용자에게 동일한 학습 계획 제공
- **진행률 추적의 어려움**: 사용자별 읽기 이력 실시간 추적 및 분석의 복잡성
- **진행률 추적 부재**: 사용자 진행 상황에 따른 실시간 추적 및 통계 제공 기능 부재
- **동기 부여 메커니즘 부재**: 연속 읽기 기록, 통계 등 학습 지속성을 높이는 기능 부족

#### 2. 원어 분석의 기술적 난제
히브리어/그리스어 등 고대 언어의 실시간 분석은 다음과 같은 기술적 어려움이 있다:

- **실시간 파싱 어려움**: 복잡한 문법 구조와 형태 변화의 실시간 처리
- **문맥 기반 의미 추출**: 단어의 다의성과 문맥에 따른 의미 변화 처리
- **문화적 배경 통합**: 고대 문화적 맥락 정보의 자동 추출 및 통합

## 발명이 해결하려는 과제

본 발명은 상기한 문제점을 해결하기 위해 다음과 같은 과제를 해결하고자 한다:

1. **개인화된 읽기 계획 생성**: 사용자별 읽기 이력, 진행률, 패턴을 실시간으로 분석하여 개인 맞춤형 학습 계획을 자동 생성하는 새로운 알고리즘 제공

2. **고대 언어 실시간 분석**: 히브리어/그리스어 텍스트를 실시간으로 파싱하고, 문법 분석, 의미 추출, 문화적 배경 정보를 통합하여 제공하는 새로운 분석 방법 제공

3. **커뮤니티 기반 학습 경험 통합**: 개인 학습과 커뮤니티 상호작용을 통합하여 사용자가 읽은 구절에 대한 느낌이나 생각을 공유하고, 다른 사용자와 토론할 수 있는 새로운 학습 플랫폼 제공

## 과제 해결 수단

### 시스템 구성

본 발명에 따른 AI 기반 성경 학습 플랫폼은 다음 구성요소를 포함한다:

#### 1. 읽기 이력 분석 모듈
- **사용자 읽기 이력 수집**: 사용자가 읽은 성경 구절, 읽은 날짜, 완료 여부 등을 실시간으로 수집
- **이력 데이터 분석**: 읽기 패턴, 진행률, 연속 읽기 기록 등을 분석
- **통계 데이터 생성**: 일별, 월별, 연도별 읽기 통계 생성

#### 2. 계획 생성 모듈
- **MacChain 읽기 계획 기반**: 365일 체계적 읽기 계획 생성 알고리즘
- **개인화 알고리즘**: 사용자 이력을 기반으로 한 맞춤형 계획 생성

#### 3. 진행률 추적 모듈
- **실시간 진행률 기록**: 사용자의 읽기 완료 여부를 실시간으로 기록
- **연속 읽기 추적**: 연속 읽기 일수 및 최장 연속 기록 관리
- **통계 집계**: 읽기 통계를 실시간으로 계산 및 저장
- **진행률 시각화**: 사용자에게 진행률을 시각적으로 제공

#### 4. 게시판 모듈
- **게시글 작성**: 사용자가 읽은 구절에 대한 느낌이나 생각을 게시글 형태로 작성
- **댓글 시스템**: 게시글에 대한 댓글 작성 및 대댓글 지원
- **상호작용 기능**: 좋아요, 북마크 기능
- **카테고리 분류**: 일반, 신학, 묵상, 질문, 원어 등 카테고리별 분류

#### 5. AI 분석 모듈
- **원어 변환**: 성경 구절을 히브리어/그리스어 원어로 변환
- **문법 분석**: 원어 텍스트의 문법 구조 분석
- **의미 추출**: 문맥 기반 의미 분석 및 추출
- **문화적 배경 통합**: 고대 문화적 맥락 정보 자동 제공
- **다양한 AI 모델 지원**: Cloudflare AI, OpenAI, Anthropic Claude 등 다양한 AI 모델로 구현 가능

#### 6. 데이터 저장 및 관리
- **데이터베이스**: 사용자 데이터, 읽기 계획, 진행률, 게시글, 댓글 등을 저장
- **다양한 DB 지원**: PostgreSQL, MySQL, MongoDB, Cloudflare D1 등 다양한 데이터베이스로 구현 가능

### 핵심 기술 상세

#### 기술 1: 개인화된 읽기 계획 생성 알고리즘

**McCheyne 읽기 계획 기반 시스템:**

1. **기본 계획 생성 알고리즘**
   ```
   - 365일 체계적 읽기 계획 생성
   - 매일 구약 2장, 신약 2장 할당
   - 날짜별 읽기 항목 사전 계산
   - 데이터베이스에 저장
   ```

2. **개인화 알고리즘**
   ```
   - 사용자 읽기 이력 분석
   - 진행률 패턴 파악
   - 개인별 맞춤형 계획 생성
   - 시작 날짜 조정 (사용자 가입일 기준)
   ```

**기술적 특징:**
- 개인별 맞춤형 학습 경로 제공
- 학습 지속성 향상
- 진행률 추적 및 통계 제공으로 동기 부여

#### 기술 2: 게시판 기능을 통한 커뮤니티 학습 경험

**게시판 기능 구현:**

1. **게시글 작성 시스템**
   ```
   - 사용자가 읽은 구절 선택
   - 구절에 대한 느낌이나 생각 작성
   - 제목, 내용, 성경 구절 참조 입력
   - 카테고리 선택 (일반, 신학, 묵상, 질문, 원어 등)
   ```

2. **상호작용 기능**
   ```
   - 댓글 작성 및 대댓글 지원
   - 좋아요 기능 (게시글 및 댓글)
   - 북마크 기능
   - 사용자 간 피드백
   ```

3. **카테고리별 분류**
   ```
   - 일반: 일반적인 성경 토론
   - 신학: 신학적 주제 토론
   - 묵상: 개인 묵상 나눔
   - 질문: 성경 관련 질문
   - 원어: 원어 분석 토론
   ```

4. **성경 구절 연동**
   ```
   - 게시글에 성경 구절 참조 자동 연결
   - 구절 본문 자동 표시
   - 원어 분석 결과 연동
   ```

**데이터베이스 스키마:**
```sql
- mccheyne_plan: 날짜별 읽기 계획
- reading_progress: 사용자별 읽기 진행률
- user_progress: 사용자 통계 (연속 읽기, 완독률 등)
- discussions: 토론 게시글
- discussion_comments: 토론 댓글
- discussion_likes: 좋아요 정보
- discussion_bookmarks: 북마크 정보
```

#### 기술 3: 원어 분석 및 의미 추출 시스템

**실시간 원어 분석 프로세스:**

1. **텍스트 입력 및 전처리**
   ```
   - 사용자로부터 성경 구절 입력
   - 구절 참조 파싱 (예: "요한복음 3:16")
   - 원어 텍스트 조회
   ```

2. **AI 기반 문법 분석**
   ```
   - 히브리어/그리스어 텍스트 토큰화
   - 단어별 품사 분석
   - 형태 변화 분석 (어미, 접두사, 접미사)
   - 문법 구조 파싱
   ```

3. **의미 추출 및 해석**
   ```
   - 문맥 기반 의미 분석
   - 다의어 해석 (문맥에 따른 의미 선택)
   - 문화적 배경 정보 통합
   - 신학적 의미 추출
   ```

4. **결과 통합 및 제공**
   ```
   - 원문 텍스트
   - 단어별 문법 분석
   - 전체 의미 해석
   - 문화적 배경 정보
   - 실용적 적용 제안
   ```

**AI 모델 활용:**
- 다양한 AI 모델 지원: Cloudflare AI, OpenAI GPT, Anthropic Claude, Google Gemini 등
- 자연어 처리: 텍스트 이해 및 생성
- 문맥 분석: 문장 및 단락 단위 의미 파악

## 발명의 효과

### 1. 개인화된 학습 경험 제공
- **맞춤형 계획**: 사용자별 읽기 이력 기반 개인화된 학습 계획
- **통계 제공**: 진행률 기반 통계 시각화 및 동기 부여
- **학습 지속성**: 연속 읽기 기록 기반 동기 부여 시스템

### 2. 커뮤니티 기반 학습 경험
- **지식 공유**: 사용자 간 느낌과 생각 공유
- **상호작용**: 댓글, 좋아요, 북마크를 통한 활발한 커뮤니티
- **카테고리별 탐색**: 관심 주제별 토론 참여

### 3. 정확한 원어 이해
- **문법 분석**: 히브리어/그리스어의 정확한 문법 분석
- **의미 추출**: 문맥 기반 의미 해석
- **문화적 배경**: 고대 문화적 맥락 정보 제공

### 4. 기술적 혁신
- **개인화 알고리즘**: McCheyne 계획 기반 개인화된 읽기 계획 생성
- **커뮤니티 통합**: 개인 학습과 커뮤니티 상호작용의 통합
- **원어 분석**: 고대 언어 실시간 분석의 정확도 향상

## 청구항

### 제1항 (독립항 - 시스템)
사용자의 읽기 이력을 실시간으로 수집 및 분석하는 읽기 이력 분석 모듈과,
상기 읽기 이력을 기반으로 개인화된 읽기 계획을 생성하는 계획 생성 모듈과,
상기 계획에 따른 진행률을 실시간으로 추적하고 통계를 제공하는 진행률 추적 모듈과,
사용자가 읽은 구절에 대한 느낌이나 생각을 공유하고 다른 사용자와 토론할 수 있는 게시판 모듈과,
성경 원어(히브리어 또는 그리스어)를 실시간으로 분석하여 문법 분석, 의미 추출, 문화적 배경 정보를 제공하는 AI 분석 모듈을 포함하는 것을 특징으로 하는 AI 기반 성경 학습 플랫폼.

### 제2항 (종속항 - AI 분석)
제1항에 있어서,
상기 AI 분석 모듈은,
입력된 성경 구절을 원어(히브리어 또는 그리스어)로 변환하는 단계와,
상기 원어 텍스트를 문법적으로 분석하는 단계와,
상기 분석 결과를 기반으로 문맥적 의미를 추출하는 단계와,
고대 문화적 배경 정보를 통합하는 단계와,
상기 의미를 사용자 언어로 번역하여 제공하는 단계를 수행하는 것을 특징으로 하는 시스템.

### 제3항 (종속항 - 개인화된 계획)
제1항에 있어서,
상기 계획 생성 모듈은,
MacChain 읽기 계획을 기반으로 한 365일 체계적 읽기 계획을 생성하는 단계와,
사용자의 읽기 이력 및 진행률을 분석하여 개인별 맞춤형 계획을 생성하는 단계를 수행하는 것을 특징으로 하는 시스템.

### 제4항 (종속항 - 게시판 기능)
제1항에 있어서,
상기 게시판 모듈은,
사용자가 읽은 성경 구절에 대한 느낌이나 생각을 게시글 형태로 작성하는 기능과,
상기 게시글에 대한 다른 사용자의 댓글 작성 기능과,
상기 게시글 및 댓글에 대한 좋아요 및 북마크 기능과,
카테고리별 분류 기능(일반, 신학, 묵상, 질문, 원어 등)을 제공하는 것을 특징으로 하는 시스템.

### 제5항 (종속항 - 진행률 추적)
제1항에 있어서,
상기 진행률 추적 모듈은,
사용자의 일일 읽기 완료 여부를 실시간으로 기록하는 단계와,
연속 읽기 일수를 추적하고 최장 연속 기록을 관리하는 단계와,
읽기 통계를 실시간으로 집계하여 시각화하여 제공하는 단계를 수행하는 것을 특징으로 하는 시스템.

### 제6항 (방법 독립항)
사용자의 읽기 이력을 수집하는 단계와,
상기 이력을 기반으로 MacChain 읽기 계획을 활용하여 개인화된 읽기 계획을 생성하는 단계와,
상기 계획에 따른 진행률을 실시간으로 추적하고 통계를 제공하는 단계와,
사용자가 읽은 구절에 대한 느낌이나 생각을 공유하는 게시판 기능을 제공하는 단계를 포함하는 것을 특징으로 하는 개인화된 성경 읽기 계획 제공 방법.

### 제7항 (방법 - 원어 분석)
사용자로부터 성경 구절을 입력받는 단계와,
상기 구절을 원어(히브리어 또는 그리스어)로 변환하는 단계와,
상기 원어 텍스트를 문법적으로 분석하는 단계와,
상기 분석 결과를 기반으로 문맥적 의미를 추출하고 고대 문화적 배경 정보를 통합하는 단계와,
상기 의미를 사용자 언어로 번역하여 제공하는 단계를 포함하는 것을 특징으로 하는 AI 기반 성경 원어 분석 방법.

### 제8항 (종속항 - 게시판 상호작용)
제6항에 있어서,
상기 게시판 기능 제공 단계는,
사용자가 작성한 게시글에 대한 다른 사용자의 댓글 작성 기능과,
상기 게시글 및 댓글에 대한 좋아요 기능과,
상기 게시글에 대한 북마크 기능을 추가로 포함하는 것을 특징으로 하는 방법.

### 제9항 (종속항 - 연속 읽기)
제6항에 있어서,
상기 방법은,
사용자의 연속 읽기 일수를 추적하는 단계와,
상기 연속 읽기 기록을 기반으로 동기 부여 메시지를 제공하는 단계를 추가로 포함하는 것을 특징으로 하는 방법.

### 제10항 (종속항 - 통계)
제6항에 있어서,
상기 방법은,
사용자별 읽기 통계를 실시간으로 집계하는 단계와,
상기 통계를 시각화하여 제공하는 단계를 추가로 포함하는 것을 특징으로 하는 방법.

## 발명을 실시하기 위한 구체적 내용

### 실시예 1: 개인화된 읽기 계획 생성

**시나리오**: 신규 사용자의 첫 읽기 계획 생성 및 개인화

1. **초기 계획 생성**
   - McCheyne 계획 기반 365일 계획 생성
   - 매일 구약 2장, 신약 2장 할당
   - 오늘 날짜부터 시작하는 계획 할당
   - 데이터베이스에 저장

2. **개인화 조정**
   - 사용자가 3일 연속 읽기 완료
   - 시스템이 빠른 진행 패턴 감지
   - 추가 읽기 항목 제안 (선택적)

3. **진행률 추적**
   - 사용자가 읽기 완료 표시
   - `reading_progress` 테이블에 기록
   - 연속 읽기 일수 자동 업데이트
   - 통계 자동 업데이트


### 실시예 2: 개인화된 읽기 계획 및 게시판 기능

**시나리오**: 신규 사용자의 첫 읽기 계획 생성 및 커뮤니티 참여

**구현 예시 (Cloudflare D1 데이터베이스 사용):**

1. **초기 계획 생성**
   - McCheyne 계획 기반 365일 계획 생성
   - 오늘 날짜부터 시작하는 계획 할당
   - 데이터베이스에 저장 (Cloudflare D1, PostgreSQL, MySQL 등)

2. **진행률 추적**
   - 사용자가 읽기 완료 표시
   - `reading_progress` 테이블에 기록
   - 통계 자동 업데이트
   - 진행률 시각화 제공

3. **게시판 기능 활용**
   - 사용자가 읽은 구절에 대한 느낌 작성
   - 토론 게시글 작성 (제목, 내용, 성경 구절 참조)
   - 다른 사용자의 게시글에 댓글 작성
   - 좋아요 및 북마크 기능 활용
   - 카테고리별 토론 탐색 (일반, 신학, 묵상, 질문, 원어 등)

**참고**: 본 실시예는 Cloudflare D1을 사용한 구현 예시이며, PostgreSQL, MySQL, MongoDB 등 다른 데이터베이스로도 동일한 방식으로 구현 가능함

### 실시예 3: 원어 분석 및 게시판 연동

**시나리오**: 사용자가 원어 분석을 요청하고 결과를 게시판에 공유

1. **원어 분석 요청**
   - 사용자가 "요한복음 3:16" 원어 분석 요청
   - AI 모델이 그리스어 원문 분석
   - 문법 분석, 의미 해석, 문화적 배경 정보 제공

2. **게시판에 공유**
   - 분석 결과를 게시판에 게시글 형태로 작성
   - 카테고리: "원어" 선택
   - 다른 사용자들이 원어 분석 결과 확인

3. **토론 및 학습**
   - 다른 사용자가 원어 분석에 대한 질문 댓글 작성
   - 전문가 사용자가 추가 설명 제공
   - 집단 지성을 통한 학습 효과
