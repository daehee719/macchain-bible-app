UI/UX 가이드라인 (MVP용 초안)

대상: 목회자, 개인 신자, 성경 스터디 리더

디자인 원칙
- 단순성: 핵심 작업(오늘 읽기, 분석 요청, 진행 체크)으로 사용자를 빠르게 안내
- 가독성: 타이포그래피 중심, 충분한 라인 간격, 색 대비 준수
- 접근성: 키보드 네비게이션, 스크린리더 라벨, 명확한 대체 텍스트

주요 화면 흐름
- 홈 대시보드
  - 오늘의 읽기 계획(카드 목록) + ‘분석’ 버튼(구절 선택 시 AI 분석 요청)
  - 빠른 체크(읽음 표시)와 진행률 요약
- 분석 화면
  - 입력(구절/텍스트) → 분석 유형 선택(신학/묵상/역사) → 분석 결과 표시(원문/해석/메모)
  - 결과 저장/공유(공개 저장소에 민감 내용 비노출)
- 통계 화면
  - 연속 일 수, 주/월 통계, 완료율 차트(간단한 바/라인)
- 계정/동기화 화면
  - 로그인/회원가입, 동기화 상태, 동기화 수동 트리거

컴포넌트 목록(우선 구현)
- Card(읽기 계획 항목)
- Button(주요 액션) — primary/secondary/ghost
- Modal(로그인, 분석 결과 상세)
- Input/TextArea(분석 입력)
- Small charts (sparkline, bar)

네이티브 고려사항(추후)
- 푸시 알림(읽기 리마인더), 로컬 캐시(오프라인 읽기), 플랫폼별 네비게이션

와이어프레임 제안
- 우선순위: 모바일 우선 반응형 디자인 → 데스크탑 확장
- 빠른 시안(그리드 기반) 생성 후 Figma 파일로 이관 권장
