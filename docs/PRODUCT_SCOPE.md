제품 범위 (MVP) — MacChain Bible App

목표: 1주일 내에 출시 가능한 웹 + 네이티브 대비 MVP를 정의한다. 우선 사용자(목회자, 개인 신자, 성경 스터디 리더)의 핵심 니즈를 빠르게 충족시키고, 특허 대상인 'AI 원어 분석'을 안전하게 포함한다.

우선 포함할 기능(우선순위 순)
- 1) 성경 읽기 계획 제공
  - McCheyne 기반의 일일 읽기 계획 조회 (`GET /api/mccheyne/today`)
  - 사용자가 진행 상태를 표시할 수 있는 최소한의 체크 기능
- 2) AI 원어 분석 (특허 대상)
  - 텍스트 입력 → Cloudflare AI 호출 → 분석 결과 반환 및 저장 (`POST /api/ai/analyze`)
  - 분석 결과 보기, 최근 분석 목록 조회 (`GET /api/ai/history`)
  - 특허 보호가 필요한 알고리즘/프롬프트/파이프라인 세부사항은 내부 문서로 분리(공개 저장소에 전문 내용 미기재)
- 3) 읽기 계획 동기화
  - 계정 생성/로그인(JWT) 및 읽기 진행 동기화(`POST /api/users/progress`)
  - 소규모 동기화 충돌 해결(최근 수정 우선)
- 4) 통계 대시보드(간략)
  - 기본 통계: 연속 일수, 월별/주별 읽기량, 완독율 (`GET /api/statistics/user`)

MVP에서 제외(후속 릴리스)
- 복잡한 소셜/그룹 기능(공동 읽기, 초대 등)
- 고급 분석 파이프라인(대량 배치 처리) — 우선 실시간/온디맨드 중심
- 완전한 오프라인 네이티브 동기화(플랫폼별 캐싱은 최소 구현)

비기능 요구
- 보안: JWT 인증, 민감 데이터(원어 분석 결과의 개인 정보)는 최소 보관 정책 적용(보존 기간 명시)
- 규정: 특허 관련 민감 데이터 및 프롬프트는 공개 저장소에 기록하지 않음
- 성능: 응답 목표 < 500ms(읽기 계획 조회), AI 응답은 모델 한계로 비동기 처리 권장

단계적 일정 (1주 스냅샷)
- Day 0: 이 문서 확정, CI/ENV 시크릿 확인
- Day 1-2: 백엔드(Workers) 엔드포인트 안정화 — 인증, 읽기 계획, 헬스
- Day 3-4: AI 분석 엔드포인트 통합 + DB 저장, 간단한 프론트엔드 뷰
- Day 5: 통계 화면 및 동기화 API 완성
- Day 6: E2E 스모크 테스트, 배포 워크플로우 점검
- Day 7: 버그 픽스, 릴리스 준비

승인자: 사용자(대희 강)
